- 属性

~~~markdown
1. 属性的分类
		1. 类属性
			1. 直接定义在类中的属性
			2. 类创建的所有对象都有对应的类属性，且属性值相同
			3. 类属性在使用的时候，不用创建一个对象
					创建对象以后也可以使用
			4. 类属性属于类
			5. 类属性变化则所有对象全部发生变化
					牵一发而动全身
			6. 可以通过：类名.新属性 = 值    来添加类属性
			7. 类属性的作用域：整个类，全类
		2. 实例属性
			1. 属于实例对象的属性---称之为实例属性
			2. 创建在方法内部的属性是实例属性
				1. self.属性名=值
				2. 实例对象.新属性名=值
			3. 实例属性属于实例对象，通过类无法调用
			4. 实例属性如果发生改变，不影响其他的实例对象，更不影响类属性
			5. 实例对象如果要使用，必须进行实例化
			6. 如果实例属性和类属性发生命名冲突，在当前实例对象范围内，遮蔽类属性
			7. 实例属性的作用范围：这整个实例对象
2. 实例：
		通过类创建的对象称之为实例
~~~

- 小结

~~~markdown
1. 实例属性：
	各个实例各自拥有，相互之间不影响，不影响类
2. 类属性：
	全类都拥有，有且只有一个值，类属性发生了更改，影响所有的实例对象（如果类属性和实例属性命名发生冲突），实例属性会覆盖。
~~~

- 方法

~~~markdown
1. 结构：
		1. 方法的声明
			def 方法名(self)：
		2. 方法的实现
			方法内容
		3. 返回值
2. 方法的使用
		通过.的形式来调用
3. 通过实例对象来调用方法
		自动传递self参数
4. 通过类去调用方法
		不会自动传递self参数
5. 方法的使用，必须创建实例对象
6. 如果方法需要传递多个参数，则使用逗号隔开
		函数中的所有参数类别，在方法中同样使用
~~~

- self

~~~markdown
1. self指的是当前的实例对象
2. 在通过实例对象调用方法的时候，python会自动的将当前的对象传递给self
		当前对象：谁调用这个方法，谁就是当前对象
3. 在通过类调用时，python不会自动传值给self
		需要手动的去传递self值，理论上来说，self可以传递任何对象，但是理论上应该传递一个该类对应的实例对象
4. self存在的意义
		区分不同的实例对象
5. self仅仅是一个普通的变量而已
6. 自动传入的当前对象，默认传递给第一个参数
		习惯上，第一个参数，写成self
~~~

- 构造方法

~~~markdown
1. __init__(self)
		构造方法---初始化方法
2. 创建时自动调用，不需要手动调用
		也可以手动调用
3. 创建对象的时候，类名(构造参数)
		构造参数实际上是传递给了构造方法
4. init方法常常用于，在创建对象的时候初始化实例属性
		创建私人订制的实例属性
5. 一个类中可以不写构造方法，但是在创建对象的时候，会调用父类的init方法
		如果没有构造方法，则调用父类的构造方法
		可以不写，不能没有
6. 构造方法的返回值，只能是None
~~~
